1.Візуал створення тікет
1.Заголовок інциденту: Вибір з переліку: Софтовий баг, Фізична поломка, Не може зарядитись, Перерахунок, Поганий зв'язок, ДВС, Зламалось авто, Інше 
(можливість додавати нові значення) 

2.Номер інциденту: Генерується автоматично при збереженні 

3.Відділ: Вибір відділу, який буде обробляти тікет (Технічна підтримка, Сервісний відділ, Білінг, NOC, Розробка) 

4.Станція: Пошук по номеру станції (1892, 2099...). Автоматично підтягується: сервісний номер (EFC001892), адреса, власник, виробник 

5.Порт станції: CCS 2, CHADEMO, GBT DC, GBT AC, Type 2 socket, Type 2 plug, Type 1, NACS DC, NACS AC 

6.Контактна інформація: Ім'я клієнта, телефон, джерело звернення 

7.Опис: Текстове поле для детального опису інциденту 

8.Вкладення: Кнопка "Додати файли" для завантаження фото, документів 

9.Логи станції: Текстове поле для OCPP логів, повідомлень клієнта. Кнопка "Розпізнати AI" для розпізнавання тексту та картинок 

2.Важливо: 
Прибрати з форми: "Додати станцію", "Додати оператора" 
Станції та партнери підтягуються автоматично з системи (кожного дня може бути додана нова станція чи партнер). 

3. Ticket Handler (список тікетів) 
У списку тікетів має бути фільтр: 
По відділам (Технічна підтримка, Сервіс, Білінг, NOC, Розробка) 
По відповідальним (призначені виконавці)
